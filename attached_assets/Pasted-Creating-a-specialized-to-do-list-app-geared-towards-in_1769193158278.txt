Creating a specialized to-do list app geared towards individuals with ADHD involves a straightforward yet innovative idea: minimal distractions and focused engagement. The core concept is to build a prototype where the user interface displays just one item at a time, minimizing clutter and allowing users to harness their focus on one task without distractions.
The unique selling point (USP) of this to-do list app is its focus mode feature. After a user inputs a task, event, or checklist item, the app transitions into focus mode. In this mode, the app emulates a coaching session by posing questions aimed at qualifying the task and visualizing its completion. This is where the user gets to reflect and plan effectively.
The app design starts with a minimalistic screen containing a single text box and a hour glass button to start a focus session. Users can enter their task, and it gets added and remains visible at the bottom. The text box is a persistent element on the screen starting in the center and then moves to bottom on subsequent screens. This ensures that users can always jot down spontaneous ideas before they slip away.
The focus mode is activated through a dedicated hourglass button that prompts users to select a pre-configured input and answer a series of purposeful questions one at a time. This is where we will use AI to enhance this feature, For the prototype, a standard question pool suffices. Questions like 'When do you want to finish?', 'What will the end state look like?', and 'How will this make you happy?' guide the user through thoughtful reflection.
Users have the option to either input responses or navigate through the questions without answering via next button. There will be  a "jump"  link that moves to next task if in queue. . There's also an 'End Focus Mode' option after which user can choose to go back to "input" home screen, let us call it the" freedom mode" or go to "work mode" where they can visualize work strategies like a 4x4 Eisenhower matrix or a sprint Kanban.  This flexibility ensures that the app adapts to the user's current needs.
Post-focus mode, a simple table is generated in "work mode" . This table contains the task alongside the user's responses to the coaching questions. Each question and corresponding response is organized in columns, providing a clear overview of the task and its qualifiers.
The prototype is to be developed using TypeScript and HTML, operating off a simple web server to demonstrate proof of concept. This initial version will serve as a foundation for a full-scale application development in the future, incorporating Elixir for the back-end, Super Base as the database, and Flutter for a refined front-end experience.
